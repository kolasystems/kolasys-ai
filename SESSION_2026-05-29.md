## Web App (kolasys-ai)

### AI Meeting Series / Intelligent Folders
| Commit | Description |
|--------|-------------|
| `1e5c039` | feat: AI meeting series — schema, detection service, tRPC router, REST endpoints, web sidebar |
| `89b146b` | feat: PATCH /api/v1/series/{id} — rename series from desktop |

**Schema:**
```prisma
model MeetingSeries {
  id           String    @id @default(cuid())
  orgId        String
  name         String
  description  String?
  autoDetected Boolean   @default(true)
  createdAt    DateTime  @default(now())
  updatedAt    DateTime  @updatedAt
}

model RecordingSeriesMembership {
  id          String   @id @default(cuid())
  seriesId    String
  recordingId String
  @@unique([seriesId, recordingId])
}
```

**Detection algorithm** (`src/services/series-detection.service.ts`):
- Runs at summarization worker step 8.45 (after AI title generation)
- Normalizes title: strips date prefix, removes punctuation, takes first 5 words
- Word overlap similarity: threshold 0.5 (50%)
- Lookback: 90 days, 50 most recent recordings
- Match found → assign to existing series or create new series with 2 recordings
- Fire-and-forget (never blocks pipeline)

**tRPC router** (`series` in root.ts):
- `series.list` — all series for org with recentMeetings[]
- `series.get` — single series with all recordings
- `series.rename` — update name, set autoDetected: false
- `series.delete` — delete series + memberships
- `series.addRecording` / `series.removeRecording`

**REST endpoints:**
- `GET /api/v1/series` — series list
- `GET /api/v1/series/{id}` — series detail
- `PATCH /api/v1/series/{id}` — rename

**Web UI:**
- SeriesNavSection in sidebar (hidden when 0 series, shows up to 5)
- `/dashboard/series/[id]` — detail page with inline-editable title
- Ask AI from series → `/dashboard/search?q=About the meeting series "{name}": {question}`

### Calendar-Bot Worker
| Commit | Description |
|--------|-------------|
| `bb4a44a` | feat: calendar-bot worker — auto-deploy Recall.ai bot 5min before calendar meetings |

**New file:** `src/workers/calendar-bot.worker.ts`

**How it works:**
1. Polls every 60 seconds
2. Finds all orgs with `autoRecordMeetings: true` + calendar tokens connected
3. Fetches events from Google Calendar (15-min lookahead) + Microsoft Calendar
4. Extracts Zoom/Teams/Meet URLs from event title/description/location
5. Deploys Recall.ai bot 4-6 minutes before meeting starts
6. Deduplication: checks for existing recording with same URL in last 30 min
7. Falls back gracefully on any error — never blocks

**New schema fields:**
```prisma
model Organization {
  autoRecordMeetings  Boolean   @default(true)
  lastCalendarBotRun  DateTime?
}
```

**Settings toggle:** "Automatically record meetings" in Settings (AutoRecordMeetingsToggle component)

**Railway service:** `calendar-bot-worker` (service ID: `049425d8-63cf-422e-98a6-3ce3b30de9ec`)
- Start command: `npx tsx src/workers/calendar-bot.worker.ts`
- 29 env vars (copied from summarization-worker)
- Region: US West (auto-assigned)
- Status: **Online** ✅ — processing ~1 poll/minute
- Deploy logs: `[calendar-bot] alive — processed N polls` + `[calendar-bot] poll complete in ~500ms`

**⚠️ Important:** `autoRecordMeetings` defaults to `true` for all orgs. Once calendar is connected, bots will auto-join automatically.

## Key Technical Notes Added

- `backdrop-filter: blur()` only works in Electron when window `backgroundColor: '#00000000'` AND body has `background: transparent`
- Deepgram temp tokens via `POST https://api.deepgram.com/v1/auth/grant` with `{ time_to_live: 60 }`
- `ws` package required in Electron for WebSocket (`npm install ws`)
- Railway CodeMirror raw editor accepts paste via `ClipboardEvent` dispatch
- Series detection threshold: 50% word overlap, 90-day window, 50-recording lookback
- Azure iOS/macOS platform auto-generates `msauth.{bundleId}://auth` redirect URI
