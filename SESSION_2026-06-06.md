## Critical Discoveries

### 1. Recall.ai webhook silently broken by automatic_video_output
Pipeline was broken from `22ce2be` (custom bot camera shipped) until `0283b69` (disabled). Every bot recording was stuck at PROCESSING because Recall.ai silently nulled `webhook_url` when the bot create payload included `automatic_video_output`. The bot joined, recorded, went to status `done` — but never fired any webhook because none was registered on the bot.

**Evidence:** Direct API check on stuck bot `6c635115` showed `webhook_url: null` despite our payload sending it.

**Open question for Recall.ai support:** Why does including `automatic_video_output` null out `webhook_url`? Once resolved, we can restore the custom bot avatar.

### 2. calendar-bot-worker had no Microsoft/Google env vars on Railway
Auto-deploy bot was polling healthy for weeks but blind to all calendar events. `makeMicrosoftCca()` returned null silently when env vars missing, `getMicrosoftEvents` returned `[]` with no error log. Polls completed in ~400ms (no API calls) instead of ~600-800ms (real Graph API calls).

**Fix:** Set MICROSOFT_CLIENT_ID, MICROSOFT_CLIENT_SECRET, MICROSOFT_TENANT_ID, GOOGLE_CLIENT_ID, GOOGLE_CLIENT_SECRET on calendar-bot-worker via railway CLI. Vercel env vars don't propagate to Railway services.

### 3. getBotMediaUrl missing video_mixed/audio_mixed fallback
The fallback chain in `meetingbot.service.ts` only checked `audio_only`, `video_only`, `video` keys — but Recall.ai returns `audio_mixed` / `video_mixed` on done bots. This blocked recovery of stuck recordings until the fix.

### 4. Simulator builds blocked by Apple SwiftUICore restriction
`npx expo run:ios` (Debug, iphonesimulator SDK) fails at app target link with "cannot link directly with 'SwiftUICore'". Root cause: ExpoModulesCore imports SwiftUI; on simulator SDK, Apple's linker resolves SwiftUI's re-export of private SwiftUICore as a direct top-level input, and SwiftUICore's allowable_clients is SwiftUI-only.

**Device Release builds unaffected** — confirmed via `xcodebuild -sdk iphoneos -configuration Release CODE_SIGNING_ALLOWED=NO build` which links cleanly. TestFlight Archive path works.

### 5. Desktop popup window invisible from LSUIElement background app
Popup BrowserWindow with `transparent: true` shown from a background app composites invisibly on macOS — the window is created, positioned, `isVisible()` returns true, but nothing paints. This made the tray click "do nothing" despite the handler firing correctly.

**Fix:** `vibrancy: 'hud'` without `transparent: true`. Opaque NSVisualEffectView always paints.

## Architecture Decisions Reinforced
- OAuth redirect URLs MUST stay split: Google = `AuthSession.makeRedirectUri()`, Microsoft = `Linking.createURL('/')`. DO NOT unify.
- Neon HTTP: no transactions, no upsert, no updateMany (use findFirst + update pattern)
- autoRecordMeetings defaults to true in schema, but existing orgs may need manual toggle
- Custom bot avatar via Recall.ai automatic_video_output requires support ticket
