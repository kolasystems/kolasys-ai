**Upload flow fixed (was silently broken):**
- `/api/upload` route doesn't exist on the web. Old `RecordScreen` was 404ing on every upload.
- Rewrote to use the correct 4-step tRPC flow:
  1. `recordings.create` → creates DB record
  2. `recordings.getUploadUrl` → gets pre-signed S3 PUT URL
  3. `FileSystem.uploadAsync(url, uri, { httpMethod: 'PUT' })` → direct S3 upload
  4. `recordings.confirmUpload` → enqueues transcription
- Uses `expo-file-system/legacy` for `FileSystem.getInfoAsync`

**Push notifications (local-only, no APNs required):**
- New `src/lib/notifications.ts` — `initNotifications()`, `detectReadyTransitions()`, `useReadyStore` (Zustand)
- `App.tsx` calls `initNotifications()` on sign-in
- `HomeScreen` polls every 20s, detects PROCESSING→READY transitions, fires `Notifications.scheduleNotificationAsync`
- Green "Notes are ready" banner on Home screen — tap to open recording, X to dismiss
- `RecordingDetailScreen` also detects READY transition during 5s polling, calls `markReady()`

**Ask AI tab:**
- New `src/components/AskAITab.tsx`
- 4th tab on `RecordingDetailScreen` (Notes / Transcript / Actions / **Ask AI**)
- Full SSE streaming via `XMLHttpRequest` (most reliable for progressive SSE in RN)
- Calls `POST https://app.kolasys.ai/api/ai/ask` with `{ messages, recordingId }`
- Maintains chat history, shows retrieved transcript-chunk sources below each reply
- Suggested questions on empty state

**API contract corrections found:**

### `c568dc6` — fix: delete and retry stuck mutations on mobile

**Root cause:** Every POST mutation on mobile was returning HTTP 400.

tRPC batch POST body must be an **object keyed by position**, not an array:
- ❌ Wrong: `JSON.stringify([{ json: input }])`
- ✅ Correct: `JSON.stringify({ '0': { json: input } })`

**Files fixed:**
- `src/lib/api.ts` — `trpcPost` body format
- `src/screens/RecordingDetailScreen.tsx` — file-local copy of `trpcPost`

**Bonus fix:** `trpcPost<T>` made generic (was `void`) so return values can be typed — needed for `retryStuck` response shape `{ success, reason }`.

**Silent casualties of this bug (now fixed):**
- Upload flow (RecordScreen) — was also using the wrong format
- Action item toggles on Tasks tab — optimistic UI masked the failed persistence
- All other POST mutations throughout the app

## Key Technical Decisions Made This Session

**Audio retention default = keep (not delete)**
Changed from "privacy by design always-delete" to "keep by default, org can opt into deletion." Rationale: re-transcribe and audio playback require the file. Privacy-conscious orgs can enable auto-delete in settings.

**tRPC batch POST body format**
`{ '0': { json: input } }` not `[{ json: input }]` — this was silently breaking ALL post mutations on mobile. Fixed in `api.ts` and `RecordingDetailScreen.tsx`.

**No APNs for push notifications**
Local notifications only — fire `Notifications.scheduleNotificationAsync` on PROCESSING→READY transition detected via polling. No server push needed, no APNs certificate required.

**Worker health checks**
Added 60-second alive logging with job counts to make Railway logs grep-able and confirm workers are processing correctly in production.
