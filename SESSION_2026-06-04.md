## Critical Fixes

### 1. Calendar bot env vars (THE blocker)
calendar-bot-worker on Railway was missing MICROSOFT_CLIENT_ID,
MICROSOFT_CLIENT_SECRET, MICROSOFT_TENANT_ID, GOOGLE_CLIENT_ID,
GOOGLE_CLIENT_SECRET. makeMicrosoftCca() returned null silently,
getMicrosoftEvents returned [] with no logging. The bot worker was
polling healthy for weeks but blind to all calendar events.

Resolution: Set all 5 vars on calendar-bot-worker via railway CLI.
Microsoft auth now succeeds, Graph API returns events.

### 2. autoRecordMeetings was false on existing orgs
Default schema is @default(true) but Paul's org was created before
that default was set. Manually toggled ON via web Settings page.

### 3. Microsoft OAuth on iOS
Original working state: Google uses AuthSession.makeRedirectUri(),
Microsoft uses Linking.createURL('/'). DO NOT unify these — they
genuinely need different redirect URLs.

### 4. Calendar disconnect HTTP 500
Neon HTTP adapter doesn't support updateMany in transactional context.
Pattern: findFirst + update.

### 5. Voice Memos transcription stalling
Default BullMQ lockDuration (30s) was too short for the full
download → re-encode → Whisper pipeline. Increased to 300s.
